# Bricked Up - MVP Implementation Plan

## Context
Build an iOS app replicating Brick's core screen time control features using personal NFC chips. The user wants to take back control of their screen time. The existing project is a fresh SwiftUI scaffold (Hello World template, Xcode 26.4, iOS 26.4).

## Top 6 MVP Features
1. **NFC Tap-to-Lock/Unlock** — physical chip toggles blocked/unblocked state
2. **App Blocking** — Shield selected apps using Apple's Screen Time API (FamilyControls/ManagedSettings)
3. **Website Blocking** — Block distracting websites in Safari and browsers via WebContentRestrictions
4. **Custom Blocking Modes** — Named profiles (Work, Sleep, Study) with different blocked apps AND websites per mode
5. **Scheduling** — Auto-activate modes at set times/days via DeviceActivity framework
6. **Usage Tracking & Streaks** — Dashboard showing bricked time, streaks, and session history

## Phase 1: NFC Tap-to-Lock/Unlock

### New Files
- `Services/NFCService.swift` — `NFCTagReaderSession` with `.iso14443` polling to read hardware UID (secure, non-clonable)
- `Services/BrickingService.swift` — Central orchestrator: validates chip, toggles state, applies/removes shields, creates sessions
- `Views/Onboarding/NFCRegistrationView.swift` — Scan + register NFC chip on first use

### Key Detail
Use `NFCISO7816Tag.identifier` (hardware UID) not NDEF content — it's burned into the chip and can't be cloned.

### Verification
- Scan NFC tag, see UID logged; register chip; re-scan and app recognizes it; state toggles on tap

## Phase 2: App & Website Blocking via Screen Time API

### New Files
- `Services/ShieldingService.swift` — Wraps `ManagedSettingsStore`; applies/removes shields per mode using named stores. Handles both app shields AND web content restrictions.
- `Views/Modes/AppSelectionView.swift` — Wraps `FamilyActivityPicker` for app selection
- `Views/Modes/WebsiteBlockListView.swift` — UI for managing blocked website domains per mode

### Extension Files (auto-generated by Xcode, then customized)
- `ShieldConfigurationExtension.swift` — Custom shield UI ("This app is bricked")
- `ShieldActionExtension.swift` — Primary action: "Stay Bricked" → `.close`

### How Website Blocking Works
- `ManagedSettingsStore` supports `store.shield.webDomains` to block specific web domains
- `FamilyActivitySelection` includes `webDomainTokens` — the FamilyActivityPicker lets users select web domains alongside apps
- Additionally, use `store.webContent.blockedByFilter` with `WebContentSettings.FilterPolicy` for URL-level blocking
- Blocked domains show the same shield overlay in Safari and any browser using `WKWebView`
- Users can also manually enter custom domains (e.g., "reddit.com", "twitter.com") which get added to the `WebDomain` block list

### Key Details
- `ManagedSettingsStore` persists across app launches/reboots. Shields remain even if app is killed — prevents bypass.
- Web domain blocking works in Safari automatically. Third-party browsers that use `WKWebView` (Chrome, Firefox, etc.) also respect `ManagedSettings` web content restrictions.
- `FamilyActivityPicker` already supports web domain selection — the same picker can select both apps and domains.

### Verification
- Select apps AND websites via FamilyActivityPicker; NFC tap blocks them all
- Blocked apps show shield overlay; blocked websites show restriction page in Safari
- Manually added domains (e.g., reddit.com) are also blocked
- Tap NFC again to unblock everything

## Phase 4: Scheduling

### New Files

- `DeviceActivityMonitorExtension.swift` — `intervalDidStart`: apply shields; `intervalDidEnd`: remove shields (reads mode data from App Group UserDefaults)

## Architecture Notes

| Concern | Approach |
|---------|----------|
| Cross-process data | App Group UserDefaults (not shared SwiftData — too complex for MVP) |
| NFC security | Hardware UID via `NFCTagReaderSession` + `.iso14443`, not NDEF |
| Store persistence | One `ManagedSettingsStore` per mode (named `mode-{id}`) |
| Concurrency | Project uses `MainActor` default isolation; mark services `nonisolated` where needed |
| Serialization | `FamilyActivitySelection` is `Codable` → JSON encode to Data |
| Testing | Must use physical device — NFC and FamilyControls don't work in simulator |

## Required Frameworks
CoreNFC, FamilyControls, ManagedSettings, DeviceActivity, SwiftData, Charts

## Implementation Order
Phase 0 first (manual Xcode setup). Then Phases 1 & 2 can be done in parallel. Then 3 → 4 → 5 → 6 sequentially.
